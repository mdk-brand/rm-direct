# rm-direct

Локальный Windows-сервис для создания черновиков рекламных кампаний в Яндекс Директе. Сервис запускается на компьютере пользователя, открывает интерфейс в браузере и работает через OAuth-токен Яндекс Директа.

## Установка

1. Откройте раздел **Releases** на странице репозитория.
2. Скачайте `rm-direct.zip` из последнего выпуска.
3. Полностью распакуйте архив в отдельную папку.
4. Запустите `rm-direct.exe`.
5. Введите логин и пароль, полученные у администратора компании.

При запуске сервис автоматически откроется в браузере и останется работать в системном трее. Для завершения работы используйте пункт **Выход** в меню значка трея.

Важно: не переносите EXE отдельно от остальных файлов. Рядом с ним должны оставаться `server.js`, `index.html`, `auth-config.json`, справочные файлы, изображения и папка `runtime`.

## Что потребуется

- Windows 10 или Windows 11;
- доступ в интернет;
- OAuth-токен Яндекс Директа;
- права на выбранный рекламный аккаунт.

Токен не записывается в файл: он передаётся локальному серверу только для выполнения запросов к API Яндекс Директа.

## Настройка внутреннего доступа

Логин и пароль не хранятся в исходниках. На основном компьютере выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\set-auth.ps1
```

Скрипт скрыто запросит пароль и создаст `auth-config.json` с отдельными солями и PBKDF2-хешами логина и пароля. Открытых учетных данных в файле не будет. Конфигурация исключена из Git, но автоматически включается в готовый ZIP при сборке Release.

После настройки снова запустите `build-exe.ps1`: конфигурация автоматически попадёт в папку `dist`. Для смены общего пароля повторно запустите `set-auth.ps1`, пересоберите и опубликуйте новую версию.

## Запуск из исходников

Установите Node.js 18 или новее и выполните:

```powershell
node server.js
```

После запуска откройте [http://127.0.0.1:8123/index.html](http://127.0.0.1:8123/index.html).

## Сборка Windows-версии

```powershell
powershell -ExecutionPolicy Bypass -File .\build-exe.ps1
```

Готовые файлы появятся в папке `dist`. Скрипт использует установленный `node.exe`; также можно заранее положить Node.js runtime в `dist\runtime`.
